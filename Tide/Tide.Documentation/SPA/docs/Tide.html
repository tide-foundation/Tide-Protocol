Tide

The client side library to interface with the Tide ecosystem.

new Tide(vendorId, serverUrl)

Initialize Tide.
Parameters:
Name Type Description
vendorId String Your designated VendorId in which you will operate
serverUrl String The endpoint of your backend Tide server

login(username, password)

Login to a previously created Tide account. The account must be fully enabled by the vendor before use. This will generate a new Tide user using the provided username and providing a keypaid to manage the account (user-secret).
Parameters:

<async> recover(username)

Send a request to the ORK nodes used by the user to email them recovery shards. This is step 1 in a 2 step process to recover the user keys.
Parameters:
Name Type Description
username String The username of the user who wishes to recover

register(username, password, email, orkIds)

Create a new Tide account. This will generate a new Tide user using the provided username and providing a keypaid to manage the account (user-secret).
Parameters:
Name Type Description
username String Plain text username of the new user
password String Plain text password of the new user
email String The recovery email to be used by the user.
orkIds Array The desired ork nodes to be used for registration. An account can only be activated when all ork nodes have confirmed they have stored the shard.
Fires:
  • event:progress
Returns:
- The users keys to be used on the data
Type
AESKey
